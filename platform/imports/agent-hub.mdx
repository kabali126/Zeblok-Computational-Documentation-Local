## Overview

The **Agent Hub** (under **Platform → Imports**) is a catalogue of reusable **MCP (Model Context Protocol) server** templates that your **Agents** *(beta)* can deploy as tools. Each entry packages an image, its protocol, ports, parameters, and the compute plans it is allowed to run on — the panel describes it as *"a reusable MCP server entry for agent deployments."* Once an entry exists here, it can be deployed repeatedly without re-entering its configuration.

The hub ships with several entries already in place (marked **Added by SuperAdmin**), and you can register your own. An entry can be sourced three ways:

- **Docker Image** — spawn a Kubernetes pod from a Docker image. Use this for self-hosted MCP servers.
- **External URL** — register an MCP server hosted elsewhere.
- **Microservice** *(beta)* — back the entry with a platform microservice.

This page documents the **Docker Image** source. The **External URL** and **Microservice** sources are covered in their own sections.

<Note>
  **Before you start:**

  - You need to be a member of the organisation the entry will belong to.
  - The compute plans you want to attach must already exist under [Configurations → Plans](https://zeblokcomputationalprivatelimited.mintlify.app/platform/configurations/plans).
  - If your Docker image lives in a private registry, have its registry credentials ready for the **Registry** tab.
  - Agent Hub is currently in **beta**.
</Note>

## The Agent Hub catalogue

Open **Platform → Imports → Agent Hub** to see every entry. The **All** tab lists all entries; the **MCP Server** tab filters to MCP server entries. Use **Search the hub** to find an entry by name, and **Filters** to narrow the list.

<Frame>
  <img src="/images/import-agent-hub/before-importing-agent.png" alt="Agent Hub catalogue showing pre-seeded MCP server entries on the All tab" />
</Frame>

Each card shows the entry's name, its protocol (and where relevant the backing service, e.g. MCP · ChromaDB), the image tag, the number of ports and environment variables, a **Server** status badge, who added it, and a **View Details** link. Entries provided with the platform are labelled **Added by SuperAdmin**; entries you create show no such label.

## Add an MCP server (Docker image source)

Click **Add to Hub** in the top-right corner. The **Add to Agent Hub** panel opens on the right with three tabs — **Details**, **Parameters**, and **Registry** — and a **Create Template** button at the bottom.

<Frame>
  <img src="/images/import-agent-hub/add-to-hub.png" alt="Add to Agent Hub panel open on the Details tab" />
</Frame>

<Steps>
  <Step title="Details">
    The **Details** tab describes the entry and where its image comes from.

    ### Name *(required)*

    Enter a unique **Name** for the entry — for example, `htmlscrapper`.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-name.png" alt="Name field filled in on the Details tab" />
    </Frame>

    ***

    ### Description *(required)*

    Add a short **Description** of what the MCP server does.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-description.png" alt="Description field filled in on the Details tab" />
    </Frame>

    ***

    ### Source

    Agent Hub can source an entry three ways — **Docker Image**, **External URL**, and **Microservice** *(beta)*. This section covers the **Docker Image** source; the other two are documented separately. Select **Docker Image** — the panel notes it will *"Spawn a Kubernetes pod from a Docker image. Use this for self-hosted MCP servers."*

    ***

    ### Docker Image *(required)*

    Enter the full registry path the platform will pull from — for example, `hub.docker.zeblok.com/zeblok/mcp/...`.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-image.png" alt="Docker Image field with the full registry path" />
    </Frame>

    ***

    ### Image Display Name *(required)*

    Enter the short tag to show on the catalogue card (this appears as the entry's **Image Tag**) — for example, `mcp/fetch:stateful`.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-image-display-name.png" alt="Image Display Name field filled in" />
    </Frame>

    ***

    ### Protocol

    Leave **Protocol** on **MCP**.

    ***

    ### Subscription Category

    Choose a **Subscription Category** — **Default**, **Rover**, **Copilot**, or **Developer**.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-category.png" alt="Subscription Category dropdown showing Default, Rover, Copilot, and Developer" />
    </Frame>

    ***

    ### Provider Info

    Enter the vendor or source of the image in **Provider Info** — for example, `zeblok`.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-provider-info.png" alt="Provider Info field filled in" />
    </Frame>

    ***

    ### License Type

    Set the **License Type** to **Open Source** or **Third Party**.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-license-type.png" alt="License Type dropdown showing Open Source and Third Party" />
    </Frame>

    ***

    ### Visibility

    Choose the **Visibility** — **Organization** makes the entry available to your whole organisation; **Private** keeps it to you.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-visibility.png" alt="Visibility dropdown showing Organization and Private" />
    </Frame>
  </Step>

  <Step title="Parameters">
    The **Parameters** tab defines how the MCP server runs.

    ### Ports

    Click **Add Port**, then set the protocol (e.g. **HTTP**), the port number (e.g. `8000`), and an identifier.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-parameters.png" alt="Parameters tab with empty ports, environment variables, and arguments sections" />
    </Frame>

    <Frame>
      <img src="/images/import-agent-hub/import-agent-port.png" alt="A port row with the port number being entered" />
    </Frame>

    Give the port an identifier (for example, `ui`).

    <Frame>
      <img src="/images/import-agent-hub/import-agent-port-identifier.png" alt="Port row with an identifier set and the environment variable Key field ready" />
    </Frame>

    ***

    ### Environment Variables *(optional)*

    Use **Add Variable** to add any key-value pairs the server needs at startup.

    ***

    ### Arguments *(optional)*

    Use **Add Argument** to pass startup arguments to the container.

    ***

    ### Volume Path

    Set the **Volume Path** for persistent storage — the default is `/data`.
  </Step>

  <Step title="Registry">
    The **Registry** tab controls credentials and the plans the entry can run on.

    ### Private Registry

    Leave **Private Registry** off for a public image. Enable it when the Docker image requires registry credentials.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-registry.png" alt="Registry tab showing the Private Registry toggle and the Plans search" />
    </Frame>

    <Warning>
      Enabling **Private Registry** means supplying registry credentials. Treat them as secrets, and redact them — along with any sensitive environment variables — before sharing screenshots.
    </Warning>

    ***

    ### Plans

    Use **Search and add plans** to attach one or more compute plans the entry is allowed to run on. Selected plans appear as chips.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-plan.png" alt="Plans search showing available compute plans to add" />
    </Frame>

    ***

    ### Default Plan

    Once plans are added, choose the **Default Plan** used when the entry is deployed.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-default-plan.png" alt="Selected plan chips with a Default Plan dropdown" />
    </Frame>
  </Step>

  <Step title="Create the template">
    Click **Create Template**. A success message confirms the entry was created, and it appears in the catalogue with the image tag and port count you configured.

    <Frame>
      <img src="/images/import-agent-hub/after-agent-imported.png" alt="Agent Hub catalogue showing the newly created entry with a success message" />
    </Frame>
  </Step>
</Steps>

## View an entry

Click **View Details** on any card to open its details panel. It shows the entry's **Type**, **Protocol**, **Visibility**, number of **Plans**, **Image Tag**, and configured **Ports**.

<Frame>
  <img src="/images/import-agent-hub/imported-agent-details.png" alt="Details panel for an entry showing type, protocol, visibility, plans, image tag, and ports" />
</Frame>

## Edit or delete an entry

From the details panel, click **Edit** to reopen the configuration as the **Edit hub entry** panel — the same Details, Parameters, and Registry tabs, pre-filled. Make your changes and click **Save Changes**.

<Frame>
  <img src="/images/import-agent-hub/imported-agent-edit.png" alt="Edit hub entry panel with the entry's existing values pre-filled" />
</Frame>

Click **Delete** to remove the entry from the hub.

<Warning>
  Before deleting an entry, make sure no agent depends on it. Removing it takes it out of the hub for future deployments.
</Warning>

## Important Rules

<Warning>
  Read these before registering entries in a shared organisation.
</Warning>

- **Visibility controls reach.** **Organization** exposes the entry to everyone in your organisation; **Private** keeps it to you. Choose deliberately.
- **Private registries mean credentials.** Only enable **Private Registry** when the image truly needs it, and keep those credentials — and any secret environment variables — out of shared screenshots.
- **Plans must exist first.** You can only attach compute plans that already exist under [Configurations → Plans](https://zeblokcomputationalprivatelimited.mintlify.app/platform/configurations/plans).
- **SuperAdmin entries are shared defaults.** Entries marked **Added by SuperAdmin** are provided with the platform; the ones you create are yours to edit and delete.
- **This covers the Docker image source only.** Registering an entry from an **External URL** or a **Microservice** is documented in its own section.
- **Agent Hub is in beta.** Behaviour may change.
