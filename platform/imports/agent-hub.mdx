## Overview

**Agent Hub** is a catalogue of reusable **MCP (Model Context Protocol)** server templates that your Agents *(beta)* deploy. Each entry is a reusable MCP server configuration — a deployable building block your agents connect to — so you define a server once and reuse it across agent deployments instead of rebuilding it each time.

You'll find Agent Hub under **Platform → Imports → Agent Hub**.

This page documents the **Docker Image** and **External URL** sources. The **Microservice** source is covered in its own section.

<Note>
  Agent Hub and Agents are in **beta**. The exact fields and layout may change as the feature evolves.
</Note>

***

## The Agent Hub catalogue

The catalogue lists every MCP server template available to your organisation. Each card shows the entry's name, protocol, image tag (or `external` tag), and a quick count of its ports and environment variables.

<Frame>
  <img src="/images/import-agent-hub/agent-hub-catalogue.png" alt="The Agent Hub catalogue showing MCP server template cards" />
</Frame>

Use the tabs at the top to switch between **All** and **MCP Server** views. **Search the hub** filters by name, and **Filters** narrows the list further. Entries seeded by the platform are marked **Added by SuperAdmin**; the ones you add show under your own account.

***

## Add an MCP server (Docker image source)

Use the **Docker Image** source to spawn a Kubernetes pod from a container image — this is the option for self-hosted MCP servers. Open the **Add to Hub** panel from the top-right of the catalogue and work through the three tabs: **Details**, **Parameters**, and **Registry**.

<Steps>
  <Step title="Open the Add to Hub panel">
    Click **Add to Hub** in the top-right of the catalogue. A side panel titled **Add to Agent Hub** opens on the **Details** tab.

    <Frame>
      <img src="/images/import-agent-hub/add-to-hub-button.png" alt="The Add to Hub button in the top-right of the Agent Hub catalogue" />
    </Frame>
  </Step>

  <Step title="Name and describe the entry">
    Enter a **Name** *(required)* — this is what shows on the card — and a **Description** *(required)* explaining what the server does.

    <Frame>
      <img src="/images/import-agent-hub/details-name-description.png" alt="The Name and Description fields on the Details tab" />
    </Frame>
  </Step>

  <Step title="Choose the source">
    Agent Hub can source an entry three ways — **Docker Image**, **External URL**, and **Microservice** *(beta)*. This section covers the **Docker Image** source; the **External URL** source is covered in the next section, and the **Microservice** source is documented separately. Select **Docker Image** — the panel notes it will *"Spawn a Kubernetes pod from a Docker image. Use this for self-hosted MCP servers."*

    <Frame>
      <img src="/images/import-agent-hub/source-docker-image.png" alt="Docker Image selected as the source on the Details tab" />
    </Frame>
  </Step>

  <Step title="Enter the image details">
    Provide the **Docker Image** *(required)* as a full registry path, for example `hub.docker.zeblok.com/zeblok/mcp/fetch:stateful`. Then set an **Image Display Name** *(required)* — the short tag shown on the card, such as `mcp/fetch:stateful`.

    <Frame>
      <img src="/images/import-agent-hub/docker-image-fields.png" alt="The Docker Image and Image Display Name fields" />
    </Frame>
  </Step>

  <Step title="Set protocol, category, and provider">
    Leave **Protocol** on **MCP**. Choose a **Subscription Category** — **Default**, **Rover**, **Copilot**, or **Developer** — and add **Provider Info** to record the vendor or source, for example `zeblok`.

    <Frame>
      <img src="/images/import-agent-hub/protocol-category-provider.png" alt="The Protocol, Subscription Category, and Provider Info fields" />
    </Frame>
  </Step>

  <Step title="Set licence and visibility">
    Pick a **License Type** — **Open Source** or **Third Party** — and a **Visibility**. **Organization** makes the entry available across your organisation; **Private** keeps it to you.

    <Frame>
      <img src="/images/import-agent-hub/license-visibility.png" alt="The License Type and Visibility fields" />
    </Frame>
  </Step>

  <Step title="Add ports on the Parameters tab">
    Switch to the **Parameters** tab. Under **Ports**, set the **Protocol** (such as HTTP), the **port** number (for example `8000`), and an **Identifier** (for example `ui`).

    <Frame>
      <img src="/images/import-agent-hub/parameters-ports.png" alt="The Ports section on the Parameters tab" />
    </Frame>
  </Step>

  <Step title="Add environment variables and arguments (optional)">
    Still on **Parameters**, add any **Environment Variables** as **Key** / **Value** pairs, and any **Arguments** the server needs. Both are optional — add them only if your image requires them.

    <Frame>
      <img src="/images/import-agent-hub/parameters-env-args.png" alt="The Environment Variables and Arguments sections" />
    </Frame>
  </Step>

  <Step title="Set the volume path">
    Set the **Volume Path** where the server persists data. The default is `/data`.

    <Frame>
      <img src="/images/import-agent-hub/parameters-volume.png" alt="The Volume Path field on the Parameters tab" />
    </Frame>
  </Step>

  <Step title="Configure the registry and plans">
    Switch to the **Registry** tab. If the image lives in a credentialed registry, enable **Private Registry** and supply the credentials. Then select the **Plans** the template can run on — the field is searchable and accepts multiple plans — and set a **Default Plan**.

    <Frame>
      <img src="/images/import-agent-hub/registry-plans.png" alt="The Private Registry toggle, Plans selector, and Default Plan on the Registry tab" />
    </Frame>
  </Step>

  <Step title="Create the template">
    Click **Create Template**. A success message confirms the entry was created, and the new card appears in the catalogue with its image tag and port count.

    <Frame>
      <img src="/images/import-agent-hub/after-agent-imported.png" alt="The newly created MCP server template in the Agent Hub catalogue" />
    </Frame>
  </Step>
</Steps>

## Add an MCP server (External URL source)

Use the **External URL** source when the MCP server is already hosted somewhere and you only need your agents to connect to it. As the panel notes, this points at a hosted MCP endpoint and *"No pod is spawned — the chat-runtime connects directly."* Because nothing is deployed, you only complete the **Details** tab — the **Parameters** and **Registry** tabs aren't needed for this source.

<Steps>
  <Step title="Open the panel and enter the basics">
    Click **Add to Hub**, and on the **Details** tab enter a **Name** and **Description**, exactly as in the Docker image method above.
  </Step>

  <Step title="Choose External URL as the source">
    Under **Source**, select **External URL**. The Docker image fields are replaced by an **External URL** field and a **Transport** option.

    <Frame>
      <img src="/images/import-agent-hub/import-agent-external-url.png" alt="Add to Agent Hub panel with External URL selected as the source" />
    </Frame>
  </Step>

  <Step title="Enter the endpoint">
    In **External URL**, enter the hosted MCP endpoint — for example, `https://mcp.deepwiki.com/sse`.

    <Frame>
      <img src="/images/import-agent-hub/enter-external-url.png" alt="External URL field with a hosted MCP endpoint entered" />
    </Frame>
  </Step>

  <Step title="Choose the transport">
    Set **Transport** to match the endpoint — **SSE** or **Streamable HTTP**.

    <Frame>
      <img src="/images/import-agent-hub/select-transport.png" alt="Transport dropdown showing SSE and Streamable HTTP" />
    </Frame>
  </Step>

  <Step title="Fill the remaining details">
    Set **Protocol**, **Subscription Category**, **Provider Info**, **License Type**, and **Visibility** the same way as in the Docker image method.
  </Step>

  <Step title="Create the template">
    Click **Create Template**. A success message confirms the entry was created, and it appears in the catalogue marked with an **external** tag instead of an image tag.

    <Frame>
      <img src="/images/import-agent-hub/after-import-via-external-url.png" alt="Agent Hub catalogue showing the new external URL entry tagged external, with a success message" />
    </Frame>
  </Step>
</Steps>

## View an entry

Click **View Details** on any card to open its details panel. It shows the entry's **Type**, **Protocol**, **Visibility**, number of **Plans**, **Image Tag**, and configured **Ports**.

<Frame>
  <img src="/images/import-agent-hub/view-details.png" alt="The View Details panel for an Agent Hub entry" />
</Frame>

***

## Edit or delete an entry

From an entry's details panel, use **Edit** to change its configuration or **Delete** to remove it from the catalogue.

Clicking **Edit** opens the **Edit hub entry** panel with the same fields you filled when creating it. Make your changes and click **Save Changes**.

<Frame>
  <img src="/images/import-agent-hub/edit-hub-entry.png" alt="The Edit hub entry panel" />
</Frame>

<Warning>
  **Delete** permanently removes the template from the catalogue. Agents that rely on it will no longer be able to deploy it. Delete an entry only when you're sure nothing depends on it.
</Warning>

***

## Important Rules

<Warning>
  - **MCP servers are deployable infrastructure.** A Docker-image entry spawns a pod on the plan you select, so choose **Plans** and a **Default Plan** that your organisation is provisioned for.
  - **Private images need registry credentials.** If you don't enable **Private Registry** and supply credentials for a credentialed image, the deployment will fail to pull.
  - **Redact screenshots and values.** Registry credentials, environment-variable values, and provider details can be sensitive — keep them out of anything you share.
  - **This page covers the Docker image and External URL sources.** Registering an entry from a **Microservice** is documented in its own section.
</Warning>
