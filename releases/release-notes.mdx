This page documents what changed in each release of Zeblok AI-MicroCloud. Updates from intermediate releases are folded into the next major release version.

<AccordionGroup>

  <Accordion title="v2.0.0 — May 2026">
    A major release that rebuilds the Launcher from the ground up, introduces KVM-based on-prem provisioning, adds AI-Adaptive Cache integration, and strengthens authentication and security across the platform.

    **SaaS — AI-MicroCloud Platform**

    - **AI-Adaptive Cache Plans:** Dedicated AI-Adaptive plans tab in the plan stepper with logo indicator and validation (GPU min 1, max 2 when AI-Adaptive is enabled).
    - **AI-Adaptive Workstation Support:** Workstations now support cache acceleration during creation and update, with restricted Flash Memory selection.
    - **Memory Offloading Fine-Tuning:** New fine-tuning workstation supports memory offloading, enabling larger models with reduced memory footprint.
    - **Inference Offloading + KV Cache:** Inference workloads can be offloaded with KV cache storage and reuse for improved performance on long sessions.
    - **Datacenter Page:** New datacenter page with detailed resource info modal and refreshed CPU/GPU/Flash Memory rendering.
    - **Workstation Configuration Page:** Centralized configuration page introduced with refreshed spawn flow.
    - **Organization Unit Structure:** Defined customer onboarding structure for cleaner multi-tenant separation.
    - **Namespace Member Management:** Edit namespace cards with user selection dropdown and member visibility for collaboration.
    - **Workstation Image Update for MS:** Running microservice instances can now be updated to new images without re-creation.
    - **Cluster Card UI:** Refreshed cluster cards with region key and additional data fields.
    - **Plan Display:** Plan tables now show datacenter name (instead of ID) for clarity.
    - **Replica/Storage Logic:** Multi-replica blocks PVC mount, and PVC-in-use blocks replica scaling.
    - **Private Registry Imports:** Standardized import flow across microservices and workstations.

    **SaaS — Inference & Observability**

    - **GenAI Key for Inference:** New access-control layer using GenAI Keys for both offloading and non-offloading inference.
    - **Authentication Layer for Inference:** Token-based authentication introduced as part of the API Key flow.
    - **Inference Utilization Dashboard:** Real-time monitoring of inference performance, including TTFT (Time To First Token) and TPOT (Time Per Output Token).
    - **Inference Metrics Storage:** Metrics persisted to enable historical analytics alongside live monitoring.
    - **AI-Adaptive SSD Monitoring:** New SSD Read/Write and Usage % visibility for AI-Adaptive Cache workloads.

    **IaaS — Launcher (Major Rebuild)**

    - **Modernized Launcher UI:** Ground-up rebuild with a redesigned, guided user experience.
    - **New Dashboard:** Stat cards, resource overview, and a recent clusters table for quick visibility.
    - **Multi-Step Cluster Wizard:** Guided wizard with node pool, GPU configuration, and SSH validation.
    - **Multi-Cloud + Bare Metal:** End-to-end cluster creation for AWS (EKS), Azure (AKS), and Bare Metal — across both Pro and Edge variants.
    - **Secondary Datacenter Linking:** Attach a secondary datacenter and link a secondary cluster to a primary cluster.
    - **Real-Time Deployment Log Streaming:** Live log streaming during deployment for visibility into setup progress.
    - **SSH-Based Health Checks:** Per-node metrics and SSH-based health checks for accurate cluster monitoring.
    - **3-Step AI-MicroCloud Request Flow:** Cleaner, guided experience for deployment requests.
    - **Live Cluster Status Indicators:** Pending, Setting Up, Configuration, Running, Failed, Deleting, and Deleted states surfaced in real time.
    - **Re-Apply Support:** Added for AWS and Azure Pro clusters.
    - **Edge Server Support:** Native support for edge servers when creating Edge clusters.
    - **Server Pool Management:** Add Servers and Select Server flows for Edge/Non-Edge Bare Metal with pagination.
    - **Unified Edge & Pro Cluster Model:** Edge and Pro clusters consolidated into a single management model.

    **IaaS — KVM Provisioning**

    - **KVM Host Provisioning:** End-to-end automated KVM host preparation from the Launcher, including pre-setup and post-setup workflows.
    - **Add KVM Host UI Flow:** New Add KVM Host page, prerequisite screen, Create KVM screen, and sidebar navigation with status tracking.
    - **Smart IP Discovery:** Automatic subnet scan to pick 1 VIP and 3 master IPs from the available pool.
    - **Resume-on-Interruption:** If users navigate away or the host reboots mid-setup, the flow can resume from the last completed step.
    - **Post-Reboot Verification:** Automatic verification of host readiness after reboot.
    - **Create VM Flow:** New Create VM experience with free IP/resource discovery and persistent VM details on the KVM list page.

    **IaaS — Edge & Cluster Infrastructure**

    - **Edge Platform GA:** Edge variant of the platform now generally available.
    - **Edge GPU Workloads:** GPU-enabled workloads now supported on Edge clusters.
    - **Edge MongoDB Compatibility:** Compatibility delivered for environments without AVX support.
    - **AI-Adaptive 3-Node Cluster:** Fully automated provisioning for 3-node AI-Adaptive Cache clusters.
    - **AI-Adaptive Standalone Cluster:** Standalone AI-Adaptive Cache cluster setup completed and shipped.
    - **NVMe Auto-Partitioning:** NVMe disks are partitioned automatically during Bare Metal setup.
    - **Multi-Cluster Domain Support:** Cluster domains are managed dynamically, enabling true multi-cluster deployments.
    - **Email Verification & Password Reset:** Email verification post-signup, license key validation page, and Forgot Password experience.

    **PaaS — Seeding & Onboarding**

    - **Default & Customized Seeding Flows:** Launcher now offers default and customized seeding paths with a dedicated log tracking screen.
    - **Selective Seeding:** Customers can pick exactly which Workstations, Microservices, Agents, and Models to seed, with clear seeded vs. non-seeded markers.
    - **Admin User Seeding:** Single admin identity associated across workstations, microservices, agents, and models, with credentials delivered via email.
    - **Initialize Seed Action:** "Initialize Seed" entry point on cluster cards, contextually enabled only when the cluster is active.
    - **Expanded Seed Catalog:** Seed catalog expanded with 10 new plans and corresponding mapping.
    - **Faster Catalog Loads:** Caching introduced on Launcher to reduce repeat catalog calls.

    **PaaS — Model Lifecycle & Storage**

    - **Multi-Backend Model Downloader:** Now supports AWS, Azure, and MinIO with dynamic storage backend detection.
    - **Validated Model Uploads:** Workflows validated for Mistral-7B and Llama 3.x (incl. 70B variants).
    - **MLflow 3.9.0:** Upgraded with refreshed model serving flow.
    - **Public CLI:** CLI is now publicly accessible, enabling programmatic platform interaction.
    - **Updated Workstation & Microservice Catalog:** Workstation images and core microservices refreshed to latest upstream versions.

    **PaaS — Agents & Inference Platform**

    - **OpenClaw Microservice:** Delivered with two deployment modes — self-hosted LLM and external API.
    - **Memory-Offload Inference Mode:** Inference deployment now supports memory offloading for larger model footprints.
    - **Upgraded Agent Backbone:** Agent backbone upgraded to Qwen3-Coder-Next FP8 (80B MoE) for more reliable workspace-instruction following.
    - **Custom Agent Personas:** Per-agent personas with skill allowlists, allowing tailored agent behavior across workspaces.
    - **MCP Server Integration:** External services can now be wired in as agent tools via MCP.
    - **API Key Mechanism:** API Key issuance and management for programmatic AI-MicroCloud and Inference access.

    **Security (Cross-Layer)**

    - **Control Plane Hardening:** Master and worker control plane hardening applied across managed clusters.
    - **Node-Level Security:** Kernel hardening, AppArmor profiles, and unattended security updates applied at the node level.
    - **ETCD Encryption:** Cluster state encrypted at rest.
    - **Auditing & Logging:** Cluster-level auditing and logging enabled for traceability and compliance.
    - **Network Policy Extensions:** Deny-all network policy extensions strengthen workload isolation.
    - **Server Lockdown:** Server-level access control restricting platform access to trusted endpoints.
    - **Role-Based Access Control:** Three-tier RBAC introduced — Admin, User, and Superadmin roles.
    - **AES-256 Backend Encryption:** AES-256 encryption applied for sensitive data at the backend layer.
    - **Password Strength Enforcement:** Strength rules enforced on all user-set credentials.

    **Documentation**

    - **New Zeblok Documentation Portal:** Public-facing documentation portal launched at https://zeblokcomputationalprivatelimited.mintlify.app/.
    - **KVM Implementation Guide:** New KVM design and process documentation for on-prem deployments.

    **Bug fixes**

    - Resolved namespace creation, Invite User, and Create User errors caused by missing organization context.
    - Stabilized model dropdown identifiers for predictable selection behavior.
    - NVMe listing restricted to valid host paths only.
    - Fixed save restriction when updating workstation images during onboarding.
    - Resolved image version inconsistency in Spawn WS dropdown.
    - Fixed "Selected Plan is not available for the Workstation" error in custom plan spawn.
    - Spawn requests now correctly reflect user-defined CPU and memory limits.
    - Volumes now auto-created via configuration, eliminating manual entry.
    - Stopped/deleted inferences no longer appear in the agent spawn list.
    - Fixed Azure cluster naming/casing inconsistencies and Azure log download.
    - Storage buckets and model-hub data now reliably attached to spawned environments.
    - Downloaded models stored in proper UTF-8 format for better compatibility.
    - Worker node addition to Bare Metal clusters now succeeds end-to-end.
    - Default seed data is no longer overwritten on service restart.
  </Accordion>
